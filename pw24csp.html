My.Computer Science.Project

Note: Some code has been repeated for the 3 different tasks, given the setup of the pre-release material.
Task 1 explanation:
An array of size 280 stores license numbers, visitors’ names, and parking details for 20 parking spaces over 14 days (20 x 14 = 280). A variable allows user input between 1 and 14, and a string variable accepts ‘Y’ or ‘N’ for additional bookings. For a specific day, the starting and ending range for parking spaces is calculated. If a parking space is available, the found variable is set to true, and the parking number is allocated using a formula. Users input vehicle numbers and visitor names. Enter ‘Y’ to clear all data or ‘N’ to retain it, clearing all data at the end of the week.

Task 2 explanation:
An array of size 280 stores license numbers, visitors’ names, and parking details for 20 spaces over 14 days (20 x 14 = 280). Users input a number between 1 and 14 and ‘Y’ or ‘N’ for additional bookings. For a specific day, the starting and ending range for parking spaces is calculated. Users specify if they need accessible parking (‘Y’ or ‘N’). If accessible parking is chosen, a space is found between the start and end range; otherwise, a general space is found between slots 6 and 20. If a space is available, the found variable is set to true, and the parking number is allocated using a formula. Users input vehicle numbers and visitor names.

Task 3 explanation:
Input a booking day between 1 to 14. Computer start range and end range. Set count variable to zero. Loop to check for free parking spaces starting at position 1 to 15. Computer start range and end range. Set count variable to zero. Loop to check for free parking spaces starting at position 6 to 20. Increment count variable if it is not free. Total countgeneral between 1 and 5.
